JavaScript's Q&A Section

JavaScript is the most efficient client-side scripting language. Client-side scripting language means it's run on the client computer's web-browsers. JavaScript is an interpreted language. It's gets executed line by line while running. When we run JS code, it is received by the browser's engine (v8) then the engine runs the source code to display output. V8 engine divided by 2 parts: HEAP and CALL STACK. JS works as following chart:
JavaScript File ==> V8(Parser==>AST ==>Conversion to Machine code ==>Machine code ==> Code Runs).

JavaScript Event Loop works as following:

Javascript is a single-threaded programming language. Javascript engine mainly divided by HEAP and CALL STACK . JS engine executes synchronous function first through CALL STACK. When the javascript engine (v8) finds an asynchronous function it passes the async/await function to the WEB API to execute it and CALL STACK keep continue execution of the sync function respectively. When WEB API executes the async/await function then sent to CALLBACK QUEUE. When the JS engine finishes sync function execution then EVENT LOOP sends the async/await function from CALLBACK QUEUE to CALL STACK for execution.
